## 🎯 What is VOLT OS?

**VOLT OS** is not a website builder. It's not a coding assistant. It's an **AI Workforce Platform** — a modular AI operating system where **7+ specialized agents** independently own every stage of the software lifecycle.

From requirements to deployment, VOLT OS orchestrates an **autonomous engineering organization** that produces production-ready code with measurable quality and full audit trails.

## 🚀 Getting Started

### Prerequisites

- Python 3.12+
- Node.js 18+
- PostgreSQL 16 (with pgvector)
- Redis 7

### Quick Start

```bash
# Clone the repository
git clone https://github.com/motherskitchenblr2/volt-os.git
cd volt-os

# Start infrastructure
docker-compose up -d

# Backend
cd backend
pip install -r requirements.txt
uvicorn src.main:app --reload

# Frontend
cd ../frontend
npm install
npm run dev
```

### API Endpoints

| Module | Endpoints | Description |
|--------|-----------|-------------|
| Plugins | 7 | Install, activate, deactivate, upgrade, remove, list, get |
| Pipelines | 6 | List, get, ready stages, complete, approve, status |
| Memory | 6 | Store, retrieve, search, forget, decisions |
| Observability | 5 | Summary, cost, latency, health, audit |
| Artifacts | 5 | Store, list, get, history, reject |

## 📐 Core Design Principles

| # | Principle | Description |
|---|-----------|-------------|
| P1 | Domain-agnostic core | Platform infrastructure knows nothing about software engineering |
| P2 | Plugin-first | Every capability is a plugin behind a stable API |
| P3 | Modular agent runtime | Agents are data (YAML), not hardcoded logic |
| P4 | Artifact-based orchestration | Structured exchange, never conversation |
| P5 | Planning before code | Pipeline enforced: Discovery → Deploy |
| P6 | Multi-level memory | Agent → Project → User → Org → Knowledge Base |
| P7 | Security throughout lifecycle | Not a final step — integrated at every stage |
| P8 | Sandboxed execution | All code runs in isolated environments |
| P9 | Full observability | Every action logged, every cost tracked |
| P10 | Quality over features | Clean foundation first, incremental addition |
